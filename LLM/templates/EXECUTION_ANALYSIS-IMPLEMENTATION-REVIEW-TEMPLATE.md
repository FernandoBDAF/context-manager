## üìö Usage Guidelines

**When to Use This Template**:
- Achievement completion review
- PLAN pause or milestone review
- External feedback received (e.g., code review)
- Quality validation needed
- Implementation status check

**How to Fill This Template**:
1. **Header**: Fill in Purpose, Date, Status, Related work, Category
2. **Status Review**: Document current state, what's implemented, what's missing
3. **Findings**: Document issues, gaps, and quality assessment
4. **Recommendations**: What to fix and what to improve
5. **Action Items**: Specific tasks to address findings (prioritized)

**Best Practices**:
- Be specific and evidence-based
- Include file locations and line numbers for code issues
- Prioritize findings by severity
- Link to related analyses and work
- Update status as review progresses
