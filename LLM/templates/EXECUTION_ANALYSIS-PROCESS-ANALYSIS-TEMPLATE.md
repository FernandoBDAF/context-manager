## üìö Usage Guidelines

**When to Use This Template**:
- Process issues identified (workflow friction)
- Performance problems found
- Methodology gap discovered
- Workflow optimization needed
- Process bottleneck analysis

**How to Fill This Template**:
1. **Header**: Fill in Purpose, Date, Status, Related work, Category
2. **Problem Statement**: Be specific about symptoms and impact
3. **Analysis**: Investigate thoroughly, identify root cause and contributing factors
4. **Recommendations**: Propose process improvements and workflow changes
5. **Implementation Plan**: Outline steps to improve the process

**Best Practices**:
- Be specific and evidence-based
- Include data and examples
- Identify root cause, not just symptoms
- Propose actionable improvements
- Link to related analyses and work
- Update status as analysis progresses

- See `LLM/guides/EXECUTION-ANALYSIS-GUIDE.md` for complete taxonomy and usage
