## üìù Approach

**Strategy**: Build practical decision framework that practitioners can use immediately without reading full taxonomy.

**Method**:

1. **Analyze Achievement 0.1 Results**
   - Review EXECUTION-TAXONOMY.md definitions
   - Identify key decision criteria
   - Extract distinguishing characteristics

2. **Design Decision Tree**
   - Start with primary question (SUBPLAN-connected?)
   - Branch into secondary criteria (size, purpose)
   - Lead to clear type recommendation

3. **Create When-to-Use Guidance**
   - 2-3 sentences per type
   - Real conditions where used
   - Quick example of each

4. **Develop Scenario Examples**
   - 10+ real-world situations
   - Walk through decision tree for each
   - Explain why that type chosen
   - Include edge cases

5. **Build Quick Reference Card**
   - Single-page format
   - Decision table
   - Type comparison
   - Printable/copyable

6. **Integrate with Existing Guide**
   - Add decision tree to EXECUTION-TAXONOMY.md
   - Cross-reference new documents
   - Maintain consistency

**Key Considerations**:

- Decision tree must be quick to understand (2-3 minutes max)
- Examples must be realistic and relatable
- Edge cases acknowledged and handled
- Integration with Achievement 0.1 taxonomy
